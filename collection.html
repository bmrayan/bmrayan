Methodology Overview

The DGA database was developed through a systematic data collection and quality assurance methodology designed to ensure reliability, comprehensiveness, and standardization. Our approach encompasses multiple data sources, rigorous preprocessing procedures, and comprehensive validation protocols to create a robust resource for transformer condition monitoring research.

Research Objective

To address the critical challenge of limited public access to high-quality transformer dissolved gas analysis (DGA) data by developing a comprehensive, standardized database that supports both traditional diagnostic methods and advanced machine learning approaches.

Quality Assurance Protocol

A comprehensive quality assurance framework ensures data reliability and consistency across all sources:

Data Cleaning Procedures

  • Format Normalization Standardization of gas concentration units to parts per million (ppm) and consistent naming conventions across all data sources.
  • Duplicate Detection Systematic identification and removal of duplicate records using multiple matching criteria including gas concentration patterns and metadata comparison.
  • Missing Value Assessment Analysis of data completeness with preservation of records containing partial measurements when diagnostic value is maintained.
  • Outlier Analysis Statistical identification of extreme values with retention of authentic measurements representing actual transformer conditions.
  • Data Integrity Verification Cross-validation of gas measurements against established DGA patterns and physical constraints.

Validation Criteria

Each record underwent comprehensive validation against multiple criteria:

  • Measurement Consistency: Gas concentrations checked for internal consistency and physical plausibility
  • Source Verification: Origin tracking and credibility assessment of data sources
  • Temporal Coherence: Verification of measurement timing and sequence where available
  • Standard Compliance: Alignment with established DGA measurement and interpretation standards

Statistical Characterization

Comprehensive statistical analysis was performed to characterize the dataset and validate its representativeness:

Descriptive Statistics

Full statistical profiling including measures of central tendency, dispersion, and distribution shape for each gas parameter. Analysis revealed high positive skewness (7.76-16.94) across all gases with concentration ranges spanning four orders of magnitude.

Correlation Analysis

Inter-gas relationship assessment demonstrating strong correlations among hydrocarbon gases (0.65-0.85), confirming established gas generation mechanisms and validating dataset authenticity.

Distribution Analysis

Comprehensive characterization of concentration distributions revealing log-normal tendencies essential for appropriate statistical modeling and algorithm development.

Quality Validation

Statistical patterns align with established DGA literature, confirming dataset representativeness and supporting its use for diagnostic algorithm development and validation.

Database Structure and Format

The final database is structured to maximize accessibility and usability for research applications:

Data Organization

  • Record Structure Each transformer record contains complete measurements for five key gases: hydrogen (H₂), methane (CH₄), ethylene (C₂H₄), ethane (C₂H₆), and acetylene (C₂H₂).
  • Fault Classification Subset of records (18 transformers) includes validated fault classifications providing supervised learning capabilities.
  • Metadata Preservation Source information and processing history maintained for transparency and traceability.
  • Multiple Formats Data available in JSON, CSV, and web-accessible formats to support diverse research tools and methodologies.

Access Methods

The database provides multiple access methods to accommodate different research needs:

  • Web Interface: Interactive query system with filtering and download capabilities
  • Direct Download: Complete dataset files in standard formats
  • API Access: Programmatic data retrieval for automated analysis workflows
  • Documentation: Comprehensive methodology and usage guidelines
